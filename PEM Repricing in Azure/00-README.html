PEM Repricing in Azure

Version: 1.0
Last Updated: January 16, 2026
Author: Andy Bratton
Status: Evaluation Phase

Overview

This documentation set evaluates the effort required to bring claim repricing functionality into the Azure PEM (Prospective Episode Manager) system. Repricing is a core capability of Contract Manager that calculates expected payment amounts for healthcare claims based on payer contracts and terms & conditions (TAC).

Key Requirements

Requirement Description
External Claim Data Claims will be received from external sources (not ClaimShop) in JSON format
Episode Association Repriced claims must be associated with PEM episodes for bundle calculations
Parity with CM Repricing results must match Contract Managerâ€™s existing pricing engine
Phased Approach Phase 1: Containerize existing logic; Phase 2: Cloud-native modernization

Date Decision Rationale
