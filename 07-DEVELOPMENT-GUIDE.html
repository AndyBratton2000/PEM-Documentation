Repository Setup

Clone Repositories

# Backend (Azure PEM)
git clone https://dev.azure.com/finthrive/azure-pem/_git/azure-pem
cd azure-pem

# Contract Manager (for frontend module)
git clone https://dev.azure.com/finthrive/contract-manager/_git/contract-manager

Backend Setup

cd azure-pem

# Restore NuGet packages
dotnet restore src/backend/PEM.sln

# Build solution
dotnet build src/backend/PEM.sln

# Run tests
dotnet test src/backend/PEM.sln

Frontend Setup (in Contract Manager)

cd contract-manager/app-cm-web-app/cm-web-app

# Install dependencies
npm ci

# Start development server
npm run start:dev

Local Development Environment

Docker Compose for Local Services

Create docker-compose.yml in the repository root:

version: '3.8'

services:
  sqlserver:
    image: mcr.microsoft.com/mssql/server:2022-latest
    container_name: pem-sql
    environment:
      - ACCEPT_EULA=Y
      - MSSQL_SA_PASSWORD=DevPassword123!
      - MSSQL_PID=Developer
    ports:
      - "1433:1433"
    volumes:
      - sqlserver-data:/var/opt/mssql

  redis:
    image: redis:7-alpine
    container_name: pem-redis
    ports:
      - "6379:6379"
    volumes:
      - redis-data:/data

  azurite:
    image: mcr.microsoft.com/azure-storage/azurite
    container_name: pem-azurite
    ports:
      - "10000:10000"  # Blob
      - "10001:10001"  # Queue
      - "10002:10002"  # Table
    volumes:
      - azurite-data:/data

  # Optional: Service Bus emulator (or use Azure)
  # servicebus:
  #   image: ...

volumes:
  sqlserver-data:
  redis-data:
  azurite-data:

Start local services:

docker-compose up -d

Frontend Configuration

Create app-settings.local.ts in the Angular project:

// app-cm-web-app/cm-web-app/projects/cm-web-app/src/app-settings.local.ts

import { GatewayType } from '@/gateway-type.enum';

export const appSettings = {
  getBaseUrl(gatewayType: GatewayType): string {
    switch (gatewayType) {
      case GatewayType.PEM:
        return 'https://localhost:7001';  // Local PEM API
      default:
        return 'https://localhost:5001';  // Other CM APIs
    }
  },
  
  features: {
    pemEnabled: true
  }
};

Coding Standards

.NET Backend Standards

Standard Guideline
Naming PascalCase for public, camelCase for private
Async Suffix async methods with Async
Nullability Enable nullable reference types
Exceptions Use domain-specific exceptions
Logging Structured logging with Serilog

Example:

public class EpisodeService : IEpisodeService
{
    private readonly ILogger<EpisodeService> _logger;
    private readonly IEpisodeRepository _repository;

    public EpisodeService(
        ILogger<EpisodeService> logger,
        IEpisodeRepository repository)
    {
        _logger = logger;
        _repository = repository;
    }

    public async Task<Episode> GetByIdAsync(
        Guid id,
        CancellationToken cancellationToken = default)
    {
        _logger.LogDebug("Getting episode {EpisodeId}", id);
        
        var episode = await _repository.GetByIdAsync(id, cancellationToken);
        
        if (episode is null)
        {
            throw new EpisodeNotFoundException(id);
        }
        
        return episode;
    }
}

Angular Frontend Standards

Standard Guideline
Change Detection ChangeDetectionStrategy.OnPush
State Management Store classes with BehaviorSubject
Subscriptions Use takeUntilDestroyed() or DestroyRef
Components Standalone or in feature modules
Imports Use path aliases (@/, @pem/)

Example:

import { Component, ChangeDetectionStrategy, inject, DestroyRef } from '@angular/core';
import { takeUntilDestroyed } from '@angular/core/rxjs-interop';
import { EpisodeStore } from './episode-store';

@Component({
  selector: 'pem-episode-list',
  templateUrl: './episode-list.component.html',
  styleUrls: ['./episode-list.component.scss'],
  changeDetection: ChangeDetectionStrategy.OnPush
})
export class EpisodeListComponent {
  private readonly destroyRef = inject(DestroyRef);
  private readonly episodeStore = inject(EpisodeStore);
  
  readonly episodes$ = this.episodeStore.episodes$;
  readonly loading$ = this.episodeStore.loading$;
  
  constructor() {
    this.episodeStore.loadEpisodes$
      .pipe(takeUntilDestroyed(this.destroyRef))
      .subscribe();
  }
  
  onEpisodeClick(episode: Episode): void {
    this.episodeStore.selectEpisode(episode.id);
  }
}

Database Migrations

Creating a Migration

cd src/backend/core/PEM.Infrastructure

dotnet ef migrations add <MigrationName> \
  --startup-project ../apis/PEM.Api.Episodes \
  --context PemDbContext

# Example
dotnet ef migrations add AddModelingScenario \
  --startup-project ../apis/PEM.Api.Episodes \
  --context PemDbContext

Applying Migrations

# Local development
dotnet ef database update \
  --startup-project ../apis/PEM.Api.Episodes \
  --context PemDbContext

# Generate SQL script for review
dotnet ef migrations script \
  --startup-project ../apis/PEM.Api.Episodes \
  --context PemDbContext \
  --output migration.sql

Working with Feature Flags

Enabling a Feature Flag Locally

Feature flags are managed in Azure App Configuration. For local development:

  1. Connect to nonprod App Configuration
  2. Toggle flags in Azure Portal
  3. Restart API to pick up changes (or wait 30s for refresh)

Creating a New Feature Flag

  1. Add flag in Azure App Configuration:
    • Name: PEM.FeatureArea.FeatureName
    • Label: Development (or environment)
  2. Use in backend:
[FeatureGate("PEM.FeatureArea.FeatureName")]
public async Task<IActionResult> NewFeature() { }
  1. Use in frontend:
<div *pemFeatureFlag="'PEM.FeatureArea.FeatureName'">
  <app-new-feature></app-new-feature>
</div>

Common Tasks

Adding a New API Endpoint

  1. Create request/response DTOs in PEM.Application
  2. Add domain logic in PEM.Domain
  3. Implement repository method in PEM.Infrastructure
  4. Add endpoint in appropriate API project
  5. Write unit and integration tests
  6. Update OpenAPI documentation

Adding a New Angular Component

  1. Generate component:
ng generate component pem/feature-name/component-name --module=pem
  1. Add route in pem-routing.module.ts
  2. Create store if needed
  3. Write unit tests
  4. Add to navigation if user-facing

Troubleshooting

Common Issues

Issue Solution
HTTPS certificate errors Run dotnet dev-certs https --trust
Docker connection refused Ensure Docker Desktop is running
SQL connection timeout Check Docker container is healthy
npm install fails Delete node_modules and package-lock.json, retry
Angular build errors Clear cache: ng cache clean
EF Core migration errors Ensure connection string is correct
