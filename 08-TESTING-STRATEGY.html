Overview

This document defines the testing strategy for Azure PEM, including test types, coverage requirements, and quality gates that must be met before code can progress through environments.

Testing Philosophy

Principle Description
Shift Left Test early and often in the development cycle
100% Coverage All code paths must be tested (mandatory quality gate)
Test Pyramid More unit tests, fewer integration tests, minimal E2E tests
Automation First Automate all tests; manual testing is exception only
Fast Feedback Tests should run quickly to enable rapid iteration

Test Types

Unit Tests

Purpose: Test individual units of code in isolation.

Scope: - Domain entities and value objects - Application services and use cases - Angular components, services, pipes - Utility functions

Framework: - Backend: xUnit, Moq, FluentAssertions - Frontend: Jasmine, Karma

Example (Backend):

public class EpisodeTests
{
    [Fact]
    public void AddClaim_WhenClaimInDateRange_ShouldAddSuccessfully()
    {
        // Arrange
        var episode = new Episode(
            definitionVersionId: Guid.NewGuid(),
            patientId: "P12345",
            triggerDate: new DateOnly(2026, 1, 15),
            preTriggerDays: 30,
            postTriggerDays: 90);
        
        var claim = new Claim(
            claimId: "CLM-001",
            claimDate: new DateOnly(2026, 1, 20),
            amount: 5000m);
        
        // Act
        episode.AddClaim(claim);
        
        // Assert
        episode.Claims.Should().ContainSingle();
        episode.TotalCharges.Should().Be(5000m);
    }
    
    [Fact]
    public void AddClaim_WhenClaimOutsideDateRange_ShouldThrow()
    {
        // Arrange
        var episode = new Episode(...);
        var claim = new Claim(
            claimId: "CLM-002",
            claimDate: new DateOnly(2027, 1, 1),  // Outside range
            amount: 1000m);
        
        // Act
        var act = () => episode.AddClaim(claim);
        
        // Assert
        act.Should().Throw<ClaimOutsideEpisodeRangeException>();
    }
}

Example (Frontend):

describe('EpisodeListComponent', () => {
  let component: EpisodeListComponent;
  let fixture: ComponentFixture<EpisodeListComponent>;
  let episodeStore: jasmine.SpyObj<EpisodeStore>;
  
  beforeEach(async () => {
    episodeStore = jasmine.createSpyObj('EpisodeStore', ['loadEpisodes']);
    episodeStore.episodes$ = new BehaviorSubject<Episode[]>([]);
    
    await TestBed.configureTestingModule({
      declarations: [EpisodeListComponent],
      providers: [
        { provide: EpisodeStore, useValue: episodeStore }
      ]
    }).compileComponents();
    
    fixture = TestBed.createComponent(EpisodeListComponent);
    component = fixture.componentInstance;
  });
  
  it('should display episodes from store', () => {
    const mockEpisodes = [
      { id: '1', episodeNumber: 'EP-001', status: 'Open' }
    ];
    episodeStore.episodes$.next(mockEpisodes);
    
    fixture.detectChanges();
    
    const rows = fixture.nativeElement.querySelectorAll('.episode-row');
    expect(rows.length).toBe(1);
  });
  
  it('should call store on episode click', () => {
    const episode = { id: '1', episodeNumber: 'EP-001' };
    
    component.onEpisodeClick(episode);
    
    expect(episodeStore.selectEpisode).toHaveBeenCalledWith('1');
  });
});

Integration Tests

Purpose: Test component integration with external dependencies.

Scope: - API endpoints with database - Service Bus message handlers - Redis caching - External API calls (mocked)

Framework: - Backend: xUnit, Testcontainers, WebApplicationFactory - Frontend: Angular TestBed with HttpClientTestingModule

Example (Backend API Test):

public class EpisodesApiTests : IClassFixture<PemWebApplicationFactory>
{
    private readonly HttpClient _client;
    private readonly PemDbContext _dbContext;
    
    public EpisodesApiTests(PemWebApplicationFactory factory)
    {
        _client = factory.CreateClient();
        _dbContext = factory.Services.GetRequiredService<PemDbContext>();
    }
    
    [Fact]
    public async Task GetEpisodes_ReturnsPagedResults()
    {
        // Arrange
        await SeedTestData();
        
        // Act
        var response = await _client.GetAsync("/api/episodes?page=1&pageSize=10");
        
        // Assert
        response.StatusCode.Should().Be(HttpStatusCode.OK);
        
        var result = await response.Content.ReadFromJsonAsync<PagedResult<EpisodeDto>>();
        result.Data.Should().HaveCount(10);
        result.Pagination.TotalItems.Should().Be(25);
    }
    
    [Fact]
    public async Task CreateEpisode_WithValidData_ReturnsCreated()
    {
        // Arrange
        var request = new CreateEpisodeRequest
        {
            DefinitionVersionId = Guid.NewGuid(),
            PatientId = "P12345",
            TriggerDate = DateOnly.FromDateTime(DateTime.Today)
        };
        
        // Act
        var response = await _client.PostAsJsonAsync("/api/episodes", request);
        
        // Assert
        response.StatusCode.Should().Be(HttpStatusCode.Created);
        
        var created = await response.Content.ReadFromJsonAsync<EpisodeDto>();
        created.EpisodeNumber.Should().StartWith("EP-");
    }
    
    [Fact]
    public async Task CreateEpisode_WithInvalidData_ReturnsBadRequest()
    {
        // Arrange
        var request = new CreateEpisodeRequest
        {
            DefinitionVersionId = Guid.Empty,  // Invalid
            PatientId = "",  // Invalid
            TriggerDate = DateOnly.FromDateTime(DateTime.Today)
        };
        
        // Act
        var response = await _client.PostAsJsonAsync("/api/episodes", request);
        
        // Assert
        response.StatusCode.Should().Be(HttpStatusCode.BadRequest);
    }
}

WebApplicationFactory Setup:

public class PemWebApplicationFactory : WebApplicationFactory<Program>
{
    protected override void ConfigureWebHost(IWebHostBuilder builder)
    {
        builder.ConfigureServices(services =>
        {
            // Replace SQL with Testcontainers
            var descriptor = services.SingleOrDefault(
                d => d.ServiceType == typeof(DbContextOptions<PemDbContext>));
            services.Remove(descriptor);
            
            services.AddDbContext<PemDbContext>(options =>
            {
                options.UseSqlServer(_sqlContainer.GetConnectionString());
            });
            
            // Replace Service Bus with in-memory
            services.AddSingleton<IServiceBus, InMemoryServiceBus>();
        });
    }
}

End-to-End (E2E) Tests

Purpose: Test critical user journeys through the full stack.

Scope: - Login â†’ Navigate â†’ Action â†’ Verify - Cross-service workflows - Real browser interactions

Framework: Playwright

Example:

// tests/e2e/specs/episode-workflow.spec.ts
import { test, expect } from '@playwright/test';

test.describe('Episode Workflow', () => {
  test.beforeEach(async ({ page }) => {
    await page.goto('/');
    await login(page, 'testuser@finthrive.com');
  });
  
  test('create and view episode', async ({ page }) => {
    // Navigate to episodes
    await page.click('text=PEM');
    await page.click('text=Episodes');
    
    // Click create
    await page.click('button:has-text("New Episode")');
    
    // Fill form
    await page.selectOption('#definition', 'Total Hip Replacement');
    await page.fill('#patientId', 'P12345');
    await page.fill('#triggerDate', '2026-01-15');
    
    // Submit
    await page.click('button:has-text("Create")');
    
    // Verify created
    await expect(page.locator('.snackbar')).toContainText('Episode created');
    await expect(page.locator('.episode-number')).toContainText('EP-');
  });
  
  test('search episodes with criteria', async ({ page }) => {
    await page.goto('/pem/episodes/criteria');
    
    // Set criteria
    await page.selectOption('#status', 'Open');
    await page.fill('#dateFrom', '2026-01-01');
    await page.fill('#dateTo', '2026-03-31');
    
    // Search
    await page.click('button:has-text("Search")');
    
    // Verify results
    await expect(page.locator('.episode-grid')).toBeVisible();
    await expect(page.locator('.episode-row')).toHaveCount.greaterThan(0);
  });
});

Performance Tests

Purpose: Ensure system meets performance requirements.

Scope: - API response times - Database query performance - Load testing

Framework: k6, Artillery, or Azure Load Testing

Requirements:

Metric Target Environment
API Response (P95) < 200ms Production
API Response (P99) < 500ms Production
Database Query < 50ms Production
Concurrent Users 500+ Production
Throughput 1000 req/min Production

Example k6 Test:

// tests/performance/episodes-load.js
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  stages: [
    { duration: '30s', target: 50 },   // Ramp up
    { duration: '1m', target: 100 },   // Peak load
    { duration: '30s', target: 0 },    // Ramp down
  ],
  thresholds: {
    http_req_duration: ['p(95)<200', 'p(99)<500'],
    http_req_failed: ['rate<0.01'],
  },
};

export default function () {
  const response = http.get('https://api-pem-uat.finthrive.com/api/episodes?page=1&pageSize=50', {
    headers: {
      'Authorization': `Bearer ${__ENV.AUTH_TOKEN}`,
      'X-Tenant-Id': __ENV.TENANT_ID,
    },
  });
  
  check(response, {
    'status is 200': (r) => r.status === 200,
    'response time OK': (r) => r.timings.duration < 200,
  });
  
  sleep(1);
}

Security Tests

Purpose: Identify security vulnerabilities.

Scope: - Static Application Security Testing (SAST) - Dependency vulnerability scanning - API security testing

Tools: - SAST: CodeQL, SonarQube - Dependencies: Dependabot, npm audit - API: OWASP ZAP (for UAT/Prod testing)

Code Coverage

Coverage Requirements

Metric Requirement Enforcement
Line Coverage 100% Pipeline fails if below
Branch Coverage 100% Pipeline fails if below
Function Coverage 100% Pipeline fails if below
Statement Coverage 100% Pipeline fails if below

Allowed Exclusions

Certain code may be excluded from coverage calculations with documented justification:

Exclusion Type Example Required Justification
Generated Code EF Core migrations, OpenAPI clients Auto-generated; source is tested
Infrastructure Bootstrapping Program.cs startup code Verified by integration tests
Third-party Wrappers Thin wrappers around libraries Library is well-tested

How to Exclude (.NET):

[ExcludeFromCodeCoverage(Justification = "Auto-generated EF Core migration")]
public partial class InitialCreate : Migration { }

How to Exclude (Angular):

/* istanbul ignore next - startup configuration only */
export function appConfigFactory() { ... }

Data Factories

public static class EpisodeFactory
{
    public static Episode Create(
        Guid? id = null,
        string? patientId = null,
        EpisodeStatus? status = null)
    {
        return new Episode
        {
            Id = id ?? Guid.NewGuid(),
            EpisodeNumber = $"EP-{DateTime.Now:yyyyMMdd}-{Random.Shared.Next(1000):D4}",
            PatientId = patientId ?? $"P{Random.Shared.Next(100000)}",
            Status = status ?? EpisodeStatus.Open,
            TriggerDate = DateOnly.FromDateTime(DateTime.Today),
            TotalCharges = 25000m,
            CreatedAt = DateTime.UtcNow
        };
    }
    
    public static IEnumerable<Episode> CreateMany(int count)
    {
        return Enumerable.Range(0, count).Select(_ => Create());
    }
}

CI/CD Integration

Pipeline Test Stages

stages:
  - stage: Build
    jobs:
      - job: BuildAndTest
        steps:
          # Unit Tests + Coverage
          - script: dotnet test --collect:"XPlat Code Coverage"
            displayName: 'Run Unit Tests'
          
          # Check Coverage Threshold
          - script: |
              reportgenerator -reports:coverage.xml -targetdir:coverage
              # Fail if below 100%
            displayName: 'Verify Coverage'
          
          # Security Scan
          - task: CodeQL@1
            displayName: 'Security Scan'
  
  - stage: Integration
    dependsOn: Build
    jobs:
      - job: IntegrationTests
        steps:
          - script: docker-compose up -d
          - script: dotnet test tests/PEM.Integration.Tests
          - script: docker-compose down
  
  - stage: E2E
    dependsOn: Integration
    condition: eq(variables['Build.SourceBranch'], 'refs/heads/main')
    jobs:
      - job: PlaywrightTests
        steps:
          - script: npx playwright test

Test Naming Conventions

Backend

MethodName_StateUnderTest_ExpectedBehavior

Examples: - AddClaim_WhenClaimInDateRange_ShouldAddSuccessfully - GetById_WhenNotFound_ShouldThrowNotFoundException - CalculateBundle_WithMultipleClaims_ShouldSumCorrectly

Frontend

'should [expected behavior] when [condition]'

Examples: - 'should display loading spinner when data is loading' - 'should navigate to detail page when row is clicked' - 'should disable submit button when form is invalid'
